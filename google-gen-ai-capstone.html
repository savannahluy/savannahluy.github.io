This project aims to automate and streamline the portrait culling process by leveraging Generative AI —more specifically, Gemini Flash 2.0. I will develop a system within a Kaggle Notebook that takes a collection of untouched portrait photographs as input and outputs a recommended selection of the top K images based on predefined quality and uniqueness criteria, mimicking the initial decision-making process of a professional portrait photographer.

Proposed Solution

This project aims to automate and streamline the portrait culling process by leveraging Generative AI. We will develop a system within a Kaggle Notebook that takes a collection of untouched portrait photographs as input and outputs a recommended selection of the top images based on predefined quality and uniqueness criteria, mimicking the initial decision-making process of a professional photographer.

Project Goals

The goal is to create a functional proof-of-concept that processes a batch of input images, uses Image Understanding with Structured Output to score them based on quality metrics, generates Embeddings to assess similarity, and applies a selection logic to identify a target number of high-scoring, visually distinct images. The final output will be a list or collection of the selected image identifiers, ready for the photographer's final review and editing.

Project Constraints

I wanted to utilize multimodal embeddings within this project but found that multimodal embeddings were not available through Google Gemini. This led me to experiment with Google Cloud's Vertex AI's multimodal embedding capabilities, however, I found it would be difficult for other end users to set up Vertex AI's API in order to test this Kaggle notebook. For simplicity, in this project, I will utilize text embeddings instead of image embeddings as a proxy.

Gen AI Capabilities

This solution will demonstrate the application of key Generative AI capabilities:

Image Understanding

A multimodal AI model will be employed to analyze the visual content of each photograph. It will assess critical quality factors such as lighting composition, sharpness/clarity, subject pose naturalness, and the perceived genuineness of expressions (e.g., smiles).

Structured Output (JSON Mode)

To enable systematic analysis and comparison, the Image Understanding model will be prompted to return its evaluation in a consistent, structured format (like JSON). This output will include scores or classifications for the different quality criteria for each image.

Embeddings

To address the requirement for non-redundancy, image embeddings (numerical vector representations) will be generated for each photo. These embeddings capture the visual essence of the images, allowing for quantitative measurement of similarity between them.

Future Improvements
Few Shot Prompting

Before prompting the model to analyze the given image dataset, I will provide examples of the expected analysis of each image.

Immediate Removal of Edge Case Images

Currently, the system does not remove images where the subject's eyes are closed, where the image is completely or black/white. This could improve the system's efficiency and accuracy, removing all unnessary images before they get further processed and scored.

Calculating Quality Score

Currently, each criteria for image quality (Lighting Balance, Clarity/Sharpness, Pose Naturalness, and Smile Genuineness) are being weighted equally, when calculating quality scores. Determining proper weights of each criteria could lead to an improved output, as certain criteria could be more important than others, i.e. Clarity/Sharpness being more important than Lighting Balance.
